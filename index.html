LISTA KOMEND

git config --global user.name ""

komenda, którą po instalacji Git'a ustala się swoją nazwę użytkownika. Utowrzony commit używa tych informacji i jest z nią związany. Opcja "--global" dokonuje tej konfiguracji jednorazowo tzn., że Git skorzysta z niej podczas każdej operacji wykonywanej przez użytkownika w systemie. (Przykład: git config --global user.name "Tomasz Kowalski")

git config --global user.email ""

to samo co w git config --global user.name "" aczkolwiek zamiast nazwy użytkownika konfiguruje się adres e-mail. (Przykład: git config --global user.email "tomasz.kowalski@gmail.com")

git config --global color.ui true

komenda, dzięki której Git automatycznie koloruje większość wyjścia.

git branch --merged

komenda, którą należy wpisać, gdy się chce zobaczyć wszystkie gałęzie przyłączone do obecnego brancha

git branch "branch" "commit hash"

komenda, która tworzy gałąź ze specyficznego commita, który znajduje się juz w innej gałęzi.

git branch -d "branch"

komenda ta słuzy do usunięcia danego lokalnego brancha, który był juz przekazany do zawartości repozytorium zdalnego i połączony z główną gałęzią.

git branch -v

komenda ta słuzy do tego, aby zobaczyć ostatnie rewizje na poszczególnych gałęziach.

git branch "name"

komenda, która tworzy nowego brancha o nazwie "name". (Przykład: git branch "nowy_branch")

git commit "filename"

komenda ta zatwierdza i zapisuje zmiany dokonane w pliku o nazwie "filename".

git commit -a -m ""

komenda ta zatwierdza zmiany. Po wykonaniu tego procesu, zawartość plików zapisywana jest jako migawka projektu w katalogu Git. Dzięki opcji -a wszystkie zmiany które nastąpiły w plikach i nie zostały dodane do poczekalni, również zostaną uwzględnione. Opcja -m dodaje wiadomość/ komentarz do commita.

git reset HEAD

komenda, której używamy, gdy zmiany, które wprowadziliśmy do commita nie są poprawne i chcielibyśmy je zmienić/ wycofać. Przywraca ostatnią wersję commita. np git reset HEAD (cofa o jeden commit)

gitk -all

przeglądarka repozytorium Git'a. Dzięki opcji "-all" ukazuje zmiany w repozytorium lub zaznaczonych commitach.

git pull "remote" "branch"

komendy tej uzywamy, jeśli zostały wprowadzone jakieś zmiany w repozytorium zdalnym i chcemy je pobrać do naszego repozytorium lokalnego.

git diff --staged

komenda ta słuzy do sprawdzenia jakie zmiany trafiły do poczekalni z istniejącym repozytorium.

git log --graph --all

ukazuje logi z commitów, -- graph ukazuje na podstawie tekstu graficzną reprezentacje historii commitów, -- all ukazuje wszystkie commity w historii

git fetch "remote" "branch"

komendy tej uzywamy, gdy chcemy uzyskać dane ze zdalnego projektu do naszej lokalnej gałęzi.

git push "remote" "branch"

komendy tej uzywamy kiedy chcemu wypchnąć swoje dokonane zmiany na zewnątrz. Uzywa się jej kiedy możesz i chcesz już podzielić się swoją pracą z innymi.

git log "branch" --not "branch"

komenda ta pokazuje wszystkie commity, które są w pierwszej gałęzi, ale nie ma ich w drugiej.

git bisect (start, bad, good, reset)

komenda, która pozwala na znalezienie jednego konkretnego commita, który jest be i psuje nam aplikację. Opcja "start" przeprowadza krok po kroku przez proces eliminacji dopóki znajdzie commit, który jest przyczyną błędu. Opcją "good" dajemy znać git bisect wizard'owi, który commit jest poprawny, a opcją "bad"- który jest błędny. Opcja "reset" kończy pracę git bisect wizard.

git add .

komenda, która służy do dodania pliku do poczekalni. Zapisuje wszystkie nowe zmiany w pliku. Po niej zazwyczaj uzywa się komendy git commit.

git remote add "remote_name" "git url"

komenda służąca do dodania nowego zdalnego repozytorium, która przyjmuje dwa argumenty: "remote_name" czyli nazwę zdalnego repozytorium oraz url do niego "git url". (Przykład: git remote add "nazwa" "https://github.com/user/repo.git")

git blame -C "filename"

komenda, która służy do znalezienia linijek kodu przeniesionych lub skopiowanych z innych plików, które były zmodyfikwane w tym samym commicie.
