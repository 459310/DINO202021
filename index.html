git config --global user.name ""
  • ustawia globalną nazwę użytkownika (w raz ustawioną już nie trzeba ingerować)
  • każdy utworzony przez użytkownika commit będzie uwzględniał tę nazwę w sobie, określając Twoją tożsamość
  • git config --global user.name "Bezimienny"

git reset HEAD
  • usuwa plik z poczekalni (gdzie pliki czekają na commit)
  • pozwala cofnąć przypadkowo wysłany do poczekalni plik i zapobiec commitowania go lub jak mamy kilka plików, gdy nie chcemy wysyłać ich jako jeden wspólny commit
  • git reset HEAD index.html

gitk -all
  • pokazuje wszystkie referencje (gałęzie, tagi itp.)
  • pozwala zrobić tzw. szybkie spojrzenie na całość projektu, przydaje się w organizacji
  • gitk -all

git branch --merged
  • tylko gałęzie których końcówki są scalone z nazwanym commitem zostaną wylistowane
  • pomaga w organizacji
  • git branch --merged

git config --global user.email ""
  • ustawia globalny e-mail użytkownika (w raz ustawionego już nie trzeba ingerować)
  • każdy utworzony przez użytkownika commit będzie uwzględniał tę nazwę w sobie, określając Twoją tożsamość
  • git config --global user.email "wrex.shepard@gmail.com"

git checkout -b "branch"
  • Podanie -b powoduje utworzenie nowej gałęzi o danej nazwie, a następnie zostaje ona sprawdzona
  • przyśpiesza pracę
  • git checkout -b kolejne_poprawki_FO7

git branch -d "branch"
  • usuwa gałąź jeśli ta jest już scalona, (jeśli nie to -D)
  • pomaga w usuwaniu niepotrzebnych zapychaczy w projekcie, błędu
  • git branch -d usun_mnie

git commit -a -m ""
  • -a dodaje do commita pliki zmodyfikowane lub usunięte, -m załącza opis co ten commit wnosi
  • funkcja pomocna, żeby nie commitować wszystkiego osobno i żeby było wiadomo co zostało zmienione
  • git commit -a -m "finalne_poprawki_dzialania_f.ekonomii"

git branch "name"
  • tworzy gałąź o danej nazwie
  • np. pomaga w dodaniu jakiejś funkcji dodatkowej do projektu, ale nie mieszając w głównym projekcie dopóki nie będzie działać
  • git branch Jak_wytresowac_smoka
